  • 自动更新结构说明
    • 整体架构说明
      • 平台为系统插件，其它各功能模块以程序插件的形式引入。后台通过模块和版本进行每个程序插件的管理，再在企业管理位置进行模块绑定。最终推送给客户或客户启动时检查当前版本，并进行更新实现。
      • 客户端-界面
      • 后台-模块管理
      • 后台-模块版本管理
      • 后台-企业模块版本绑定
    • 命令
      • 启动/重启:npm run start
        停止监听socket,停止框架，重新启动/upgrade.js监听，/upgrade.js 里做两个事，一个监听，一个是更新检测，如果有更新，则直接更新，不启动，如果没有更新，则启动eggjs程序.
        start已经被改造​，原start==egg-scripts start --daemon --title=jkpt_platform --workers=1 --port 7000 --framework=midway --ts --sticky
        注:不加--daemon参数，可以启动时，直接观察报错情况​
      • 停止:npm run stop
      • 编译:npm run build
        • 异常整理:
          • 会将ts文件转成js，文件路径是/dist ，混合开发和模板会不过去，会导致编译报错。需要通过直接指定/src避免出错
          • egg 部分文件声明错误问题， 通过添加//@ts-ignore忽略
      • 启动检测:npm run start_test
        非守护进程模式启动，如果启动不正常，通过这句命令可以观察到启动具体报了什么错误。
      • 安装检测:node install.js
        用于测试手动更新，避免插件未检查清楚的情况下，推送到客户端，导致启动失败。通过这句命令，可以观察到错误提示。
    • 更新流程
      • 接口平台后台主动推送/启动检查更新
      • 从后台读取各插件版本列表，对比/config.json 判断是否进行更新
        • 新插件/版本号变更的插件, 下载指定插件，解压，编译
        • 取消插件，删除本地代码，配置，编译
      • 重新启动
